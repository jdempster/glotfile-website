# Screenshot source

Reproducible demo data for the product screenshots used on the site
(`assets/screenshots/*.png`). Not part of the deployed site — exclude this
folder from the production build.

The screenshots are the **real glotfile web UI** in **dark mode**, captured at a
1512-wide viewport (retina ⇒ 3024×1746 PNGs), against a fabricated "Acme Drive"
file-storage SaaS dataset: 7 locales incl. Arabic (RTL) and Polish, plural keys,
and a deliberate spread of review states.

## Files

- `glotfile.json` — the demo dataset (generated by `gen-demo-data.mjs`).
- `gen-demo-data.mjs` — regenerates `glotfile.json` from a compact spec. Edit the
  spec here, not the JSON.
- `.glotfile/log.jsonl` — seeded Activity feed (AI batches, edits, config, etc.).
- `src/` — dummy app files so the codebase scan populates the Usage panel
  (3 files · 16 refs). The `*.tsx` reference keys via `t('…')`.

## Regenerate the screenshots

From a checkout where the `glotfile` repo is a sibling (`../glotfile`, built):

```sh
# 1. copy this folder somewhere writable and (re)generate the dataset
cp -R screenshot-source /tmp/gf-demo && cd /tmp/gf-demo
node gen-demo-data.mjs            # rewrites glotfile.json
node ../../glotfile/bin/glotfile.js export   # clears stale-output warnings in Analytics

# 2. serve the built UI against it (auto-scans cwd for the Usage panel)
node ../../glotfile/bin/glotfile.js serve -f ./glotfile.json --no-open
```

Then open the printed `http://127.0.0.1:<port>`, switch the theme to **Dark**
(sidebar, bottom-left), and capture each view at a 1512-wide viewport:

| View | Hash route | Saved as |
|------|------------|----------|
| Editor | `#editor` | `assets/screenshots/editor.png` |
| Analytics | `#analytics` | `assets/screenshots/analytics.png` |
| Activity | `#activity` | `assets/screenshots/activity.png` |
| Glossary | `#glossary` | `assets/screenshots/glossary.png` |
| Settings | `#settings` | `assets/screenshots/settings.png` |

Notes:
- Run `export` *before* screenshotting Analytics, or the "Can I ship?" panel
  fills with "outputs out of date" noise.
- The theme is stored per-origin (localStorage), so re-set Dark if the port changes.
